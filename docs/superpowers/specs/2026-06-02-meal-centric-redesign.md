# 食物营养计算器 · 餐次中心改版设计

**日期**: 2026-06-02  
**目标**: 提升记录效率 + 视觉升级 + 数据扩充

---

## 1. 背景与目标

### 当前痛点
- **记录步骤太多**：搜索 → 选食物 → 调重量 → 确认，四步流程繁琐，重复食物每次都要重搜
- **份量输入不直观**：用户想选「1碗饭」，但界面没有把份量放在主位
- **食物数据不够**：缺少奶茶品牌、零食、调味酱料等日常高频食物
- **界面风格不满意**：当前 iOS 浅绿不够好看

### 目标
- 常吃食物 **一键记录**，新食物 **两步完成**
- 按餐次（早餐/午餐/晚餐/加餐）组织今日页面，更贴合实际吃饭习惯
- 换装森系配色，视觉焕新
- 扩充 40+ 食物，覆盖品牌奶茶、零食、酱料

---

## 2. 核心改动

### 2.1 今日页：餐次中心布局

**当前**：环形图 → 营养素条 → 添加按钮 → 扁平食物列表
**改为**：紧凑环形图+营养素 → 4个餐次卡片（早餐/午餐/晚餐/加餐）

- 环形图缩小至 80-100px，与营养素进度条并排，减少垂直空间
- 每个餐次卡片：左侧彩色竖条 + 时段标签 + 食物标签列表 + 「+ 添加」按钮
- 食物以圆角小标签显示，点击标签可删除
- 餐次标题显示该时段累计 kcal
- 用户可以手动将食物标记到不同餐次

### 2.2 添加流程：两步完成 + 最近常吃

**当前**：搜索 → 下拉选食物 → 调重量 → 点确认按钮
**改为**：打开面板 → 看到「最近常吃」→ 点一下即添加；或 搜食物 → 点份量 → 直接添加

- 面板顶部新增「🔥 最近常吃」横滚区（显示最近 5-8 个常用食物，按频次排序）
- 份量选择改为 2×2 大网格按钮（半份/1份/2份/3份），代替原来的一排小按钮
- 有份量单位的食物：**点份量按钮 = 直接添加**，免去按确认按钮
- 无份量单位的食物：保留手动输入克数 + 确认按钮
- 手动克数输入降级为小行（备选方案）
- 营养预览保留（实时更新）

## 3. 数据结构改动

### 食物记录新增字段
```js
{
  // ...现有字段
  meal: 'breakfast' | 'lunch' | 'dinner' | 'snack'  // 餐次标签
}
```

### 新增 localStorage key
```js
nc2_freq_<name>: [{ foodName, count }]  // 食物频次，用于「最近常吃」
```

### 餐次默认分配规则
- 用户点在哪个餐次卡片上点「+ 添加」，新记录自动归入该餐次
- 如果没有指定（通过底部按钮添加），按时间段自动判断：
  - 5:00-10:00 → 早餐
  - 10:00-14:00 → 午餐
  - 14:00-17:00 → 加餐
  - 17:00-22:00 → 晚餐
  - 其他 → 加餐

## 4. 技术约束

- 单文件 HTML，所有改动在 `index.html` 内完成
- 向后兼容：现有 localStorage 数据可正常读取，旧记录 meal 字段默认按时间段补填
- 响应式：移动端优先，max-width 480px 居中
- 动画保留：卡片入场、环形图过渡、面板弹出
